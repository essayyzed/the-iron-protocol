## üöÄ The Three Paths

### Path A: Industry Job
- Apply to 50-100+ positions
- Practice LeetCode + System Design
- Nail behavioral interviews
- Target: Junior Developer, SWE, DevOps, Data Analyst, etc.

### Path B: Master's Program
- Research programs (GRE/TOEFL if needed)
- Statement of Purpose
- Recommendation letters
- Target: MS CS, MS Data Science, MS AI, etc.

### Path C: Entrepreneurship/Freelance
- Launch your product
- Build freelance portfolio
- Upwork/Fiverr profiles
- Indie hacker path

**Most students choose Path A ‚Üí build experience ‚Üí Path B/C later.**

## ‚ö†Ô∏è Common Year 4 Mistakes

### ‚ùå **Weak FYP topic**
Choose something impressive that showcases real skills, not a basic CRUD app.

### ‚ùå **Delaying career prep**
Start in Semester 7, not Semester 8. You need time for iterations.

### ‚ùå **Generic CV**
"Worked on a project" doesn't cut it. Quantify value: "Built X that achieved Y."

### ‚ùå **Not applying broadly**
Apply to everything you're 60%+ qualified for. Rejections are normal.

### ‚ùå **Waiting for "perfect" opportunity**
Take the first good offer. Learn. Switch later if needed.

## üí° Pro Tips for Year 4

1. **Treat job search like a project** (track applications, follow-ups, outcomes)
2. **Tailor each application slightly** (mention company/role specifics)
3. **Practice mock interviews weekly** (peers, seniors, online platforms)
4. **Leverage alumni network** (reach out on LinkedIn)
5. **Stay consistent** (apply 5-10 jobs per week minimum)
6. **Don't panic** (everyone struggles, keep applying)
