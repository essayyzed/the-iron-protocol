## üìÑ CV Best Practices

### Structure (1-2 pages max)
1. **Header** (Name, Email, LinkedIn, GitHub, Portfolio)
2. **Summary** (2-3 lines, optional but strong)
3. **Experience** (internships, projects, work)
4. **Education** (degree, university, CGPA if >3.0)
5. **Skills** (categorized: Languages, Frameworks, Tools)
6. **Projects** (3-5 best, with links)

### The "You Attitude"
Frame everything around value YOU provide:

**Bad:**
- "Worked on a web app"
- "Learned React"
- "Helped with testing"

**Good:**
- "Built CRUD web app serving 100+ users, reducing data entry time by 50%"
- "Developed responsive frontend using React, improving load time by 30%"
- "Implemented automated testing suite, catching 40+ bugs before production"

### Quantify Everything
- Number of users
- Performance improvements (%)
- Team size
- Lines of code (if impressive)
- Time saved
- Features shipped

### Tech Stack Clarity
For each project:
```
Project Name | Tech Stack | Link
Built X using Y and Z. Achieved [outcome].
```

## üíº LinkedIn Optimization

### Profile Photo
- Professional headshot
- Good lighting
- Neutral background
- Smile!

### Headline
Don't just say "Student at X University"

**Better:**
- "CS Student | Full-Stack Developer | Open Source Contributor"
- "Software Engineering Student | Specializing in Cloud & DevOps"
- "AI/ML Enthusiast | Building [Project] | Python & TensorFlow"

### About Section
Write 3-4 paragraphs:
1. Who you are + what you're passionate about
2. What you're skilled in (specific tech)
3. What you're working on now
4. How to reach you

### Experience
Treat each project/internship like a job:
- **Title** (e.g., "Full-Stack Developer (Personal Project)")
- **Date range**
- **Bullet points** using action verbs
- **Quantify** impact

### Skills & Endorsements
- List 20-30 relevant skills
- Ask peers to endorse you
- Endorse others (reciprocity)

### Recommendations
Ask for recommendations from:
- Professors (for projects)
- Internship supervisors
- Team project members
- Hackathon teammates

## üéØ Application Strategy

### Volume Matters
- **Target:** 50-100+ applications
- Apply if you're 60%+ qualified
- Don't self-reject

### Where to Apply
- **Job boards:** LinkedIn, Indeed, Glassdoor
- **Tech-specific:** AngelList, Stack Overflow Jobs, Dice
- **Company websites:** Direct applications
- **Referrals:** LinkedIn connections, alumni

### Application Tracking
Use a spreadsheet:

| Company | Role | Applied | Status | Follow-Up | Notes |
|---------|------|---------|--------|-----------|-------|
| Company A | SWE | 01-Jan | Interview | 15-Jan | Phone screen |

## üß™ Interview Prep

### Technical Interviews

#### Data Structures & Algorithms
- Practice: LeetCode, HackerRank
- Focus on: Medium difficulty
- Patterns: Two pointers, sliding window, BFS/DFS, DP

#### System Design (for senior roles)
- Learn: Scalability, load balancing, caching
- Resources: Grokking System Design, System Design Primer

#### Language-Specific
- Know your primary language deeply
- Understand OOP principles
- Be ready to write code on whiteboard/screen

### Behavioral Interviews

#### Common Questions
- "Tell me about yourself"
- "Why do you want to work here?"
- "Tell me about a time you failed"
- "Describe a conflict in a team"
- "What's your biggest weakness?"

#### STAR Method
- **Situation:** Context
- **Task:** What needed to be done
- **Action:** What YOU did
- **Result:** Outcome (quantify if possible)

#### Prepare 5-7 Stories
Cover different scenarios:
- Leadership
- Failure/Learning
- Teamwork
- Conflict resolution
- Technical challenge

### Mock Interviews
- Practice with: Peers, seniors, Pramp, interviewing.io
- Frequency: Weekly minimum
- Record yourself and review

## üö´ Common Mistakes

### ‚ùå **Waiting for "perfect" match**
Apply if you're 60%+ qualified. Learn the rest.

### ‚ùå **Generic applications**

### ‚ùå **Not following up**
