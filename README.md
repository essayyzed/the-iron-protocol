## ðŸ“– Abstract

**The Iron Protocol** is a comprehensive, semester-by-semester execution plan for Computer Science, Software Engineering, and IT students. 

It rejects the modern trend of "Vibe Coding" and passive tutorial consumption. Instead, it enforces a strict regimen of **Fundamentals First**, **Linux Literacy**, and **Systems Thinking**.

**Goal:** To graduate not just with a degree, but as an industry-ready engineer who can build, deploy, and scale complex systems.

## âš–ï¸ The Core Directives (RFC 2026)

All participants of The Iron Protocol must adhere to the following immutable laws:

1. **NO MAGIC (The AI Clause):** You MUST NOT use AI (ChatGPT/Copilot) to generate code you cannot write yourself. AI is a tutor, not a writer.
2. **CLOSE TO THE METAL:** You MUST use the Command Line Interface (CLI) for file management, git, and compilation. GUIs are a luxury.
3. **ACTIVE CREATION:** For every 1 hour of consumption (video/reading), you MUST spend 3 hours in creation (coding).
4. **THE LAW OF STRUGGLE:** When encountering an error, you MUST struggle with it for at least 20 minutes before seeking external help.
